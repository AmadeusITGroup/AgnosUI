Accordion

  • Examples
  • Api
  • Playground

Props

className

Type: string
CSS classes to be applied on the widget main container
Default value: ''

closeOthers

Type: boolean
If `true`, only one item at the time can stay open.
Default value: false

itemAnimation

Type: boolean
If `true`, accordion-item will be animated.
It is a prop of the accordion-item.
Default value: true

itemBodyClass

Type: string
Classes to add on the accordion-item body DOM element.
It is a prop of the accordion-item.
Default value: ''

itemButtonClass

Type: string
Classes to add on the accordion-item toggle button DOM element.
It is a prop of the accordion-item.
Default value: ''

itemClass

Type: string
Classes to add on the accordion-item DOM element.
It is a prop of the accordion-item.
Default value: ''

itemCollapseClass

Type: string
Classes to add on the accordion-item collapse DOM element.
It is a prop of the accordion-item.
Default value: ''

itemDestroyOnHide

Type: boolean
If `true`, the content of the accordion-item collapse will be removed from the DOM. It will be just hidden otherwise.
It is a prop of the accordion-item.
Default value: true

itemDisabled

Type: boolean
If `true`, the accordion-item will be disabled. It will not react to user's clicks, but still will be possible to toggle programmatically.
It is a prop of the accordion-item.
Default value: false

itemHeaderClass

Type: string
Classes to add on the accordion-item header DOM element.
It is a prop of the accordion-item.
Default value: ''

itemHeadingTag

Type: string
The html tag to use for the accordion-item-header.
It is a prop of the accordion-item.
Default value: ''

itemId

Type: string
The id of the accordion-item. It can be used for controlling the accordion-item via the accordion api.
It is a prop of the accordion-item.
Default value: ''

itemTransition

Type: TransitionFn
The transition to use for the accordion-item collapse when is toggled.
It is a prop of the accordion-item.
Default value: collapseVerticalTransition

itemVisible

Type: boolean
If `true`, the accordion-item will be visible (expanded). Otherwise, it will be hidden (collapsed).
It is a prop of the accordion-item.
Default value: false

onHidden

Type: (itemId: string) => void
An event fired when an item is hidden.
Event payload is the id of the item.
Default value: 
noop

onItemHidden

Type: () => void
An event fired when an item is hidden.
It is a prop of the accordion-item.
Default value: 
noop

onItemShown

Type: () => void
An event fired when an item is shown.
It is a prop of the accordion-item.
Default value: 
noop

onItemVisibleChange

Type: (visible: boolean) => void
An event fired when the `visible` value changes.
Event payload is the new value of visible.
It is a prop of the accordion-item.
Default value: 
noop

onShown

Type: (itemId: string) => void
An event fired when an item is shown.
Event payload is the id of the item.
Default value: 
noop

slotItemBody

Type: SlotContent<AccordionItemContext>
Content present in the accordion body.
It is a prop of the accordion-item.
Default value: undefined

slotItemHeader

Type: SlotContent<AccordionItemContext>
Content present in the accordion button inside the accordion header.
It is a prop of the accordion-item.
Default value: undefined

slotItemStructure

Type: SlotContent<AccordionItemContext>
Structure of the accordion-item. The default item structure is: accordion-item contains accordion header and accordion collapse; the accordion header contains the accordion button (that contains `slotItemHeader`), while the accordion collapse contains the accordion body (that contains slotItemBody).
It is a prop of the accordion-item.
Default value: undefined

Api

collapse

Type: (itemId: string) => void
Given the itemId, will collapse the corresponding accordion-item.
If the itemId is not valid, nothing will happen.

collapseAll

Type: () => void
It will collapse all the accordion-items in the accordion.

expand

Type: (itemId: string) => void
Given the itemId, will expand the corresponding accordion-item.
If the itemId is not valid, nothing will happen.

expandAll

Type: () => void
It will expand all the items in the accordion.
If `closeOthers` is `true` it will expand only the last accordion-item.

isExpanded

Type: (itemId: string) => boolean
Given the itemId, it will return if such item is visible (expanded) or not.
If the itemId is not a valid id it will return `false`.

registerItem

Type: (itemConfig?: PropsConfig<AccordionItemProps>) => AccordionItemWidget
Creates a new accordionItem.

toggle

Type: (itemId: string) => void
Given the itemId, will toggle the corresponding accordion-item.
If the itemId is not valid, nothing will happen.
